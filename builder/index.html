Link reconciliation

This page has four working sections that can be used together or independently:
  1. Find a field — search field names, isVersionOf terms, and optionally definitions, usage comments, and examples to discover which DwC-DP table contains the information you need.
  2. Select tables — choose the tables needed to represent your dataset completely. Table enabling and disabling is driven by required dependencies in the published schemas.
  3. Recommended table mapping order — generate a dependency-aware order in which to map your data into the selected tables.
  4. Generate outputs — inspect and copy a datapackage.json, and download a bundle containing the datapackage.json plus empty CSV files with headers for the selected tables.
  5. Share your configuration — the Copy Link button captures the currently selected tables in the page URL so the exact state of the tool can be shared with others. Opening that link will restore the same table selections (adjusted automatically if the schemas have changed).

Find a field

Use this section when you know a term, part of a term, or only the meaning of the data you have. Search results tell you which table contains a field, and the Enable button will check that table together with all of the required dependency tables it needs. This is often the fastest way to decide what to select in the table section below. Type at least 3 characters to search. Results update while typing.

Select tables

Use this section to build the full set of DwC-DP tables needed for your dataset. Start with at least one table from the first column of tables with no required dependencies. Only tables whose required dependencies are already selected can be checked. The field search above can help you identify which tables you need, and selections made here control what appears in the mapping-order and output sections below.

Recommended table mapping order

Use this section after selecting tables. It generates a recommended order for mapping data into the selected tables based on all foreign-key relationships among them. This does not change the selected tables, it only helps you plan the sequence of mapping work.

Generate outputs

Use this section after selecting tables. Generate a datapackage.json for inspection and copying, or download a bundle containing the datapackage.json plus one empty CSV file with headers for each selected table. The outputs always reflect the current table selection.
